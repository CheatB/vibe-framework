# Визуальная разметка workflow (обязательно к выполнению)

## Зачем это нужно
Пользователь должен ВИДЕТЬ на каком этапе workflow находится Claude.
Каждый этап начинается с визуального маркера в формате ниже.

## Маркеры этапов

### При получении задачи — определить режим:
```
━━━ 🎯 РЕЖИМ: [Quick Fix / Feature / Epic] ━━━
Задача: [краткое описание]
Оценка: ~[время]
```

### Фаза 1 — Brainstorming (Feature/Epic):
```
━━━ 💡 ФАЗА 1: BRAINSTORMING ━━━
Анализирую задачу, готовлю вопросы...
```

### Фаза 1.8 — Бизнес-анализ (Feature/Epic):
```
━━━ 💼 ФАЗА 1.8: БИЗНЕС-АНАЛИЗ ━━━
Блоки: [N/5] | Legal | Monetization | GTM | Analytics | A11y
Текущий: [название блока]
```

### Фаза 2 — Планирование (Feature/Epic):
```
━━━ 📋 ФАЗА 2: ПЛАН ━━━
Блоков: [N] | Задач: [M] | Оценка: ~[время]
[список блоков]
⏸️ Жду одобрения плана
```

### Фаза 3 — Выполнение:
```
━━━ 🔨 ФАЗА 3: ВЫПОЛНЕНИЕ ━━━
Блок [N/total]: [название блока]
```

### Между блоками — чекпоинт:
```
━━━ ✅ ЧЕКПОИНТ: Блок [N] завершён ━━━
Сделано: [что сделано]
Следующий: Блок [N+1] — [название]
⏸️ Продолжаем?
```

### Фаза 4 — Тестирование:
```
━━━ 🧪 ФАЗА 4: ТЕСТИРОВАНИЕ ━━━
Тесты: [passed/failed/skipped]
Покрытие: [X%]
```

### Фаза 5 — Коммит и деплой:
```
━━━ 📦 ФАЗА 5: КОММИТ + ДЕПЛОЙ ━━━
Коммит: [сообщение]
Push: [статус]
CI/CD: [статус]
```

### Auto-Fix Pipeline (когда срабатывает):
```
━━━ 🔧 AUTO-FIX: Уровень [1-4] ━━━
Ошибка: [описание]
Попытка: [N/3]
Действие: [что делаю]
```

### Конец работы:
```
━━━ 🏁 ГОТОВО ━━━
Сделано: [список]
Коммиты: [N шт.]
Время: ~[время]
```

## Правила использования маркеров

1. Маркер ОБЯЗАТЕЛЕН в начале каждого этапа
2. Quick Fix пропускает фазы 1-1.8-2, начинает с 🔨
3. Чекпоинт между блоками — ВСЕГДА (не пропускать)
4. Auto-Fix маркер показывается КАЖДЫЙ РАЗ при ошибке
5. Если Agent Team активен, team-lead показывает маркеры
6. Маркеры НЕ дублируются — один раз на начало фазы
