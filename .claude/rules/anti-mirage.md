# Anti-Mirage Check (Skeptic)

**ТРИГГЕР:** После написания кода >20 строк или создания tech-spec/tasks.

Claude регулярно ссылается на несуществующие файлы, функции, API, зависимости.
Это "миражи" — самая частая причина ошибок при AI-разработке.

## Когда проверять

### После написания кода (>20 строк)
1. **Импорты:** Каждый `import`/`require`/`from` ссылается на СУЩЕСТВУЮЩИЙ файл
   - Проверить: `ls <путь>` или `find . -name "<файл>"`
   - Частая ошибка: `from modules/payment import ...` — а модуля `payment` нет
2. **Вызовы функций:** Каждая вызываемая функция СУЩЕСТВУЕТ в импортируемом модуле
   - Проверить: `grep -n "def <функция>\|function <функция>" <файл>`
3. **Переменные окружения:** Каждая `os.getenv()` / `process.env.` ОПИСАНА в `.env.example`
4. **Зависимости:** Каждый внешний пакет УКАЗАН в `package.json` / `requirements.txt` / `pyproject.toml`
5. **API endpoints:** Если вызываешь внешний API — URL и метод АКТУАЛЬНЫ

### После создания tech-spec
1. Все упомянутые файлы/модули проекта существуют (или явно помечены как "создать")
2. Все внешние библиотеки существуют и не deprecated
3. Все API endpoints сторонних сервисов актуальны
4. Версии зависимостей совместимы между собой

### После создания tasks.md
1. Файлы в секции "Файлы: создать/изменить" — существуют (или задача на создание идёт раньше)
2. Зависимости между задачами непротиворечивы (нет циклов)
3. Все упомянутые модули/функции существуют в кодовой базе

## Как проверять

**Молча.** Если нашёл мираж — исправить без показа пользователю.
Исключение: если исправление меняет архитектуру или логику — спросить.

## Примеры миражей

```python
# Мираж: файл не существует
from app.services.analytics import AnalyticsService  # analytics.py не создан!

# Мираж: функция не существует в модуле
user = UserService.get_by_telegram_id(telegram_id)  # метод называется get_by_tg_id!

# Мираж: пакет не установлен
import aioredis  # не в requirements.txt!

# Мираж: env var не описана
REDIS_URL = os.getenv("REDIS_URL")  # нет в .env.example!
```

## Интеграция

Этот чек выполняется:
- Автоматически после написания кода (часть quality-gates)
- Как шаг в DoD каждого блока
- При валидации tech-spec и tasks.md
