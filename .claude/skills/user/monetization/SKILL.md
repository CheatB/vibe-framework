# Monetization

Скилл для определения модели монетизации, ценообразования, выбора платёжной системы и расчёта unit economics.

## Когда использовать

- Фаза 1.8.2 в `/new-project` pipeline
- Standalone через `/business-analysis`
- При добавлении платежей в существующий проект

## Workflow

```
1. Получить контекст → тип продукта, аудитория, конкуренты
2. Задать уточняющие вопросы
3. Предложить 2-3 модели с плюсами/минусами
4. После выбора модели:
   a. Сформировать структуру тарифов
   b. Рассчитать unit economics (базово)
   c. Рекомендовать платёжную систему
   d. Создать задачи на интеграцию
5. Записать результат в docs/business-analysis.md → секция "Monetization"
```

## Уточняющие вопросы

1. Как планируешь зарабатывать? (или «пока не знаю» — тогда AI предложит)
2. Какой средний чек у конкурентов?
3. B2B или B2C? (влияет на модель и ценообразование)
4. Готов ли MVP быть бесплатным для валидации?
5. Какая география платежей? (РФ / международные)

## 8 моделей монетизации

| Модель | Когда подходит | Пример |
|--------|---------------|--------|
| **Подписка (SaaS)** | Регулярное использование, высокая retention | Notion, ChatGPT Plus |
| **Freemium** | Массовый продукт, вирусный рост | Telegram Premium, Spotify |
| **Разовая оплата** | Инфопродукты, инструменты, курсы | Шаблоны, плагины |
| **Pay-per-use** | API, генерация контента, утилиты | OpenAI API, облачные сервисы |
| **Рекламная** | Бесплатный продукт с трафиком | Telegram-каналы, медиа |
| **Партнёрская (affiliate)** | Агрегаторы, рекомендательные сервисы | Кэшбэк-сервисы |
| **Донаты** | Контент-проекты, open-source | Boosty, GitHub Sponsors |
| **Маркетплейс (комиссия)** | Платформа между продавцом и покупателем | Авито, Ozon |

### Рекомендации по типу продукта

| Тип продукта | Рекомендуемые модели |
|---|---|
| Telegram-бот | Freemium + подписка, Pay-per-use |
| SaaS | Подписка, Freemium |
| Инфопродукт | Разовая оплата, подписка на контент |
| API-сервис | Pay-per-use, подписка с лимитами |
| Маркетплейс | Комиссия с транзакций |
| Open-source | Донаты, Freemium (hosted версия) |

## Выбор платёжной системы (decision tree)

```
Только РФ?
├── Да → ЮKassa (бывш. Яндекс.Касса)
│   ├── ИП/ООО? → да → полный функционал
│   └── Самозанятый? → ограничения, рассмотреть CloudPayments
├── РФ + СНГ → ЮKassa + Robokassa
└── Международно → Stripe + ЮKassa (dual)
    └── Stripe недоступен из РФ? → Paddle / LemonSqueezy / CloudPayments

Telegram-бот?
├── Telegram Payments API (Stars) — встроенные платежи
├── + внешняя платёжка для подписок (webhook)
└── TON-платежи (крипто, опционально)

Подписки (рекуррентные)?
├── Да → нужен recurring billing
│   ├── ЮKassa (автоплатежи) — комиссия ~3.5%
│   ├── CloudPayments (рекуррентные) — комиссия ~2.7-3.5%
│   └── Stripe Billing (международные) — комиссия 2.9% + 30¢
└── Нет → одноразовые платежи (любая система)
```

### Сравнение платёжных систем

| Система | Комиссия | Подписки | Самозанятый | Международные |
|---|---|---|---|---|
| ЮKassa | 3.5% | ✅ | ⚠️ ограничения | ❌ |
| CloudPayments | 2.7-3.5% | ✅ | ✅ | ⚠️ СНГ |
| Stripe | 2.9%+30¢ | ✅ | ❌ | ✅ |
| Telegram Stars | 0% (Apple/Google берут свои) | ❌ | ✅ | ✅ |
| Robokassa | 3.5-5% | ✅ | ✅ | ⚠️ СНГ |

## Unit Economics (базовый расчёт)

AI помогает рассчитать:

| Метрика | Формула | Ориентир |
|---|---|---|
| **CAC** (Customer Acquisition Cost) | Затраты на маркетинг / Новые клиенты | Зависит от ниши |
| **LTV** (Lifetime Value) | ARPU × Средний срок жизни клиента | — |
| **LTV/CAC** | LTV / CAC | **>3** для устойчивого бизнеса |
| **Payback period** | CAC / Ежемесячный доход с клиента | **<12 мес** |
| **MRR** (Monthly Recurring Revenue) | Подписчики × Средний чек | Для SaaS |
| **Churn rate** | Ушедшие / Всего клиентов × 100% | **<5%** для SaaS |
| **ARPU** (Average Revenue Per User) | Выручка / Активные пользователи | — |

### Пример расчёта

```
Модель: Freemium + подписка $9.99/мес
Пользователи: 1000 free, 50 paid (конверсия 5%)

MRR = 50 × $9.99 = $499.50
CAC = $5 (органика + контент)
LTV = $9.99 × 6 мес (средний срок) = $59.94
LTV/CAC = 59.94 / 5 = 11.99 ✅ (>3)
Payback = 5 / 9.99 = 0.5 мес ✅
```

## Пример вывода

```markdown
## Monetization

### Выбранная модель: Freemium + подписка
- Free: базовый функционал (до 10 запросов/день)
- Pro: 990₽/мес — безлимит + приоритет
- Team: 2990₽/мес — до 5 пользователей + API

### Ценообразование
- Средний чек конкурентов: 500-1500₽/мес
- Позиционирование: средний сегмент (990₽)
- Обоснование: ниже конкурентов на старте для захвата доли

### Unit Economics (прогноз)
- CAC: ~300₽ (органика + контент-маркетинг)
- Конверсия free→paid: 5%
- LTV: 5940₽ (средний срок подписки 6 мес)
- LTV/CAC: 19.8 ✅

### Платёжная система: ЮKassa
- Причина: только РФ, ИП, подписки нужны
- Комиссия: 3.5%
- Автоплатежи: да
- 54-ФЗ: чеки генерируются автоматически

### Задачи на интеграцию (→ tasks.md)
- [ ] Подключить ЮKassa API
- [ ] Реализовать webhook для подтверждения платежей
- [ ] Создать страницу тарифов
- [ ] Реализовать trial-период (14 дней)
- [ ] Настроить автоматическое продление
- [ ] Добавить email-уведомления о платежах
```

## Приоритет при конфликтах

1. Юридические требования (из legal-compliance) важнее удобства интеграции
2. Если конверсия free→paid < 2% → пересмотреть ценообразование или free-лимиты
3. Если LTV/CAC < 3 → бизнес-модель нежизнеспособна, нужна корректировка
4. При сомнениях в выборе — рекомендуй начать с более простой модели и усложнять по мере роста
