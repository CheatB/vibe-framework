# Legal Compliance (РФ)

Скилл для определения юридических требований к продукту в рамках российского законодательства.

## Когда использовать

- Фаза 1.8.1 в `/new-project` pipeline
- Standalone через `/business-analysis`
- При добавлении платежей, регистрации или сбора данных в существующий проект

## Workflow

```
1. Получить контекст → тип продукта, аудитория, данные
2. Задать уточняющие вопросы (батч по 3-5)
3. Определить применимые законы
4. Сформировать адаптивный чеклист
5. Дать рекомендации с учётом особенностей продукта
6. Записать результат в docs/business-analysis.md → секция "Legal Compliance"
```

## Уточняющие вопросы (задать пользователю)

1. Какие персональные данные собираются? (email, телефон, ФИО, фото, геолокация, платёжные данные)
2. Есть ли регистрация/авторизация?
3. Есть ли платежи? Какие суммы? Подписки или разовые?
4. Есть ли рассылки (email/push/SMS)?
5. Работа с несовершеннолетними (<18 лет)?
6. Есть ли UGC (контент от пользователей — комментарии, фото, видео)?
7. Open-source? Нужна ли лицензия?
8. Есть ли реклама или маркировка рекламных материалов?

## Адаптивный чеклист

### Обязательный минимум (ВСЕГДА)

- [ ] **Политика обработки персональных данных** (152-ФЗ)
  - Какие данные собираем, цели обработки, сроки хранения, права субъекта
  - Разместить: `/privacy` или `/policy`, ссылка в футере
  - Для Telegram-бота: согласие через inline-кнопку при `/start`
- [ ] **Согласие на обработку ПД** (форма)
  - Чекбокс при регистрации, текст согласия
  - Должно быть ОТДЕЛЬНЫМ от других согласий

### Если есть платежи

- [ ] **Публичная оферта** (ст. 437 ГК РФ)
  - Предмет, цена, порядок оплаты, возврата, ответственность сторон
  - Разместить: `/offer` или `/terms`
- [ ] **Соответствие 54-ФЗ** (онлайн-кассы)
  - ИП/ООО → нужна касса + ОФД + чеки покупателю
  - Самозанятый → кассу можно не использовать (до лимита 2.4 млн/год)
  - Решение: ЮKassa/CloudPayments сами генерируют чеки (проверь тариф)
- [ ] **Политика возврата средств**
  - Сроки, условия, процедура возврата

### Если есть рассылки

- [ ] **Согласие на рассылку** (отдельное от ПД!)
- [ ] **Механизм отписки** в каждом сообщении
- [ ] **Соответствие 38-ФЗ «О рекламе»**
  - Пометка «реклама», указание рекламодателя

### Если есть UGC

- [ ] **Правила сообщества / модерации**
- [ ] **Механизм жалоб на контент**
- [ ] **Ответственность за контент** (149-ФЗ)
  - Модерация + хранение данных 6 мес (для соцсетей >500К)

### Если open-source

- [ ] **Выбор лицензии**
  - MIT — максимальная свобода, минимум обязательств
  - Apache 2.0 — как MIT + защита патентов
  - GPL — copyleft, производные тоже GPL
  - Проприетарная — коммерческий продукт

### Если работа с детьми (<18 лет)

- [ ] **436-ФЗ** (защита детей от вредной информации)
- [ ] **Возрастная маркировка** (0+, 6+, 12+, 16+, 18+)
- [ ] **Согласие родителей** на обработку ПД ребёнка

### Если маркировка рекламы

- [ ] **Токены и дисклеймеры** (ОРД)
- [ ] **Пометка «реклама»** с указанием рекламодателя

## Особенности по типу продукта

### Telegram-бот
- Согласие на ПД: inline-кнопка при первом `/start`
- Оферта: ссылка в описании бота + команда `/terms`
- Платежи через Telegram Stars: упрощённая схема (Apple/Google обрабатывают)
- Внешние платежи (ЮKassa): полная схема 54-ФЗ

### SaaS / Веб-приложение
- Cookie-баннер (если используются cookie)
- Политика конфиденциальности в футере
- Оферта при регистрации
- Серверы в РФ → требование локализации (ч. 5 ст. 18 152-ФЗ) выполнено

### Инфопродукт / Курс
- Оферта обязательна (описание программы, формат, доступ)
- Политика возврата (7 дней на дистанционные покупки)
- Лицензия на образовательную деятельность (если >3 курсов, формальное обучение)

## Пример вывода

```markdown
## Legal Compliance

### Обязательные документы
1. **Политика обработки ПД** — необходима, т.к. собираются: email, имя, телефон
   - Рекомендация: разместить по ссылке /privacy, ссылка в футере
   - Особенность: данные хранятся на VPS в РФ — требование локализации выполнено
2. **Публичная оферта** — необходима, т.к. есть платежи
   - Рекомендация: /offer, ссылка в футере и при оплате

### Рекомендации
- Для Telegram-бота: согласие на обработку ПД через inline-кнопку при первом /start
- Используется ЮKassa → чеки генерируются автоматически (54-ФЗ выполнен)

### Особенности законодательства
- ⚠️ С 01.09.2025 ужесточены требования к обработке биометрических данных
- ⚠️ Если проект использует AI для обработки ПД — рекомендуется отдельное согласие
```

## Приоритет при конфликтах

1. Безопасность пользовательских данных — всегда приоритет
2. Если не уверен в юридическом вопросе — рекомендуй консультацию юриста
3. Лучше перебдеть: включи пункт в чеклист, если есть сомнения
4. Этот скилл НЕ заменяет юридическую консультацию — это чеклист для технического специалиста

## Актуальность

Законодательство: РФ, актуально на 2025-2026.
Ключевые законы: 152-ФЗ, 437 ГК РФ, 54-ФЗ, 38-ФЗ, 436-ФЗ, 149-ФЗ.
