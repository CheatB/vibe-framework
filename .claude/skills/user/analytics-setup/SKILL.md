# Analytics Setup

Скилл для определения ключевых метрик продукта, выбора инструмента аналитики, проектирования дашборда и настройки алертов.

## Когда использовать

- Фаза 1.8.4 в `/new-project` pipeline
- Standalone через `/business-analysis`
- При добавлении аналитики в существующий проект

## Входные данные

- Тип продукта (из Brainstorm)
- Модель монетизации (из блока Monetization)
- Каналы привлечения (из блока GTM)

## Workflow

```
1. Получить контекст → тип продукта, монетизация, каналы
2. Задать уточняющие вопросы
3. Определить метрики по AARRR framework
4. Рекомендовать инструмент аналитики
5. Спроектировать структуру дашборда
6. Определить алерты
7. Создать задачи на интеграцию
8. Записать результат в docs/business-analysis.md → секция "Analytics"
```

## Уточняющие вопросы

1. Что считаешь главным показателем успеха? (North Star Metric)
2. Какие действия пользователей самые важные?
3. Нужен ли self-hosted? (privacy concerns)
4. Бюджет на аналитику? (бесплатно / до $50/мес / без ограничений)

## AARRR Framework (Pirate Metrics)

| Этап | Что измеряем | Примеры метрик |
|------|-------------|---------------|
| **Acquisition** | Откуда приходят | Визиты, источники трафика, CTR рекламы, CAC по каналам |
| **Activation** | Первый «вау-момент» | Регистрация, первое действие, onboarding completion, time to first value |
| **Retention** | Возвращаются ли | DAU/MAU, D1/D7/D30, session frequency, feature usage |
| **Revenue** | Платят ли | Конверсия free→paid, ARPU, MRR, churn rate, LTV |
| **Referral** | Рекомендуют ли | Invite rate, viral coefficient, NPS |

### Метрики по типу продукта

| Тип | North Star | Ключевые метрики |
|---|---|---|
| Telegram-бот | DAU | /start rate, D7 retention, команды/сессия |
| SaaS | MAU | Onboarding %, feature adoption, churn |
| Маркетплейс | GMV (объём транзакций) | Листинги, конверсия, средний чек |
| Инфопродукт | Completion rate | Покупки, прохождение, NPS |
| API-сервис | API calls/day | Latency P95, error rate, usage growth |

## Выбор инструмента (decision tree)

```
Self-hosted нужен?
├── Да, есть VPS → PostHog (полный набор) или Umami (лёгкий)
├── Нет → облачные решения
└── Не уверен → PostHog Cloud (бесплатный тир до 1М событий)

Тип продукта?
├── Веб-приложение → PostHog / Umami / Яндекс.Метрика
├── Telegram-бот → кастомные метрики в БД + Grafana
├── Мобилка → Firebase Analytics / Amplitude
├── API/SaaS → PostHog + custom events
└── Лендинг → Яндекс.Метрика (вебвизор!) + GA4

Бюджет?
├── Бесплатно → Umami (self-hosted) / Яндекс.Метрика / GA4
├── До $50/мес → PostHog Cloud / Mixpanel free tier
└── Больше → PostHog / Amplitude / Mixpanel
```

### Сравнение инструментов

| Инструмент | Цена | Self-hosted | Фичи | Для кого |
|---|---|---|---|---|
| **PostHog** | Free до 1М / $0.00045 за событие | ✅ | Полный стек | SaaS, веб-приложения |
| **Umami** | Free (open-source) | ✅ | Базовая аналитика | Лендинги, простые сайты |
| **Яндекс.Метрика** | Free | ❌ | Вебвизор, карты | РФ-проекты, лендинги |
| **GA4** | Free до 10М | ❌ | Стандартная аналитика | Универсальный |
| **Amplitude** | Free до 10М | ❌ | Product analytics | Продуктовые команды |
| **Grafana** | Free (open-source) | ✅ | Дашборды, алерты | Серверы, боты, API |

## Структура дашборда

```
📊 Дашборд продукта

┌─────────────────────────────────────────┐
│  OVERVIEW (главный экран)               │
│  DAU | WAU | MAU | Retention D7         │
│  Revenue (MRR) | New users today        │
│  ▼ Тренд за 30 дней (график)           │
├─────────────────────────────────────────┤
│  ACQUISITION                            │
│  Источники трафика | CTR по каналам     │
│  CAC по каналам | Конверсия лендинга    │
├─────────────────────────────────────────┤
│  ACTIVATION                             │
│  Onboarding completion rate             │
│  Time to first value | Drop-off points  │
├─────────────────────────────────────────┤
│  REVENUE                                │
│  Конверсия free→paid | ARPU | Churn     │
│  MRR growth | LTV/CAC                   │
├─────────────────────────────────────────┤
│  RETENTION                              │
│  Cohort analysis | Feature usage        │
│  Session duration | Return frequency    │
└─────────────────────────────────────────┘
```

## Алерты

| Метрика | Триггер | Действие |
|---------|---------|----------|
| DAU | Падение >20% за день | Уведомление в Telegram |
| Error rate | >5% за час | Уведомление + проверка логов |
| Conversion | Падение >30% за неделю | Анализ воронки |
| Churn | Рост >10% за месяц | Ревью продукта |
| Server response | >2 сек (P95) | Оптимизация |
| Disk usage | >80% | Уведомление + cleanup |

## Пример вывода

```markdown
## Analytics

### Ключевые метрики
- **North Star**: MAU (активные пользователи в месяц)
- **Acquisition**: регистрации/день, CAC, источники
- **Activation**: % прошедших onboarding, time to first value
- **Retention**: D1=60%, D7=40%, D30=20% (таргеты)
- **Revenue**: конверсия free→paid >5%, MRR, churn <5%

### Инструмент: PostHog (self-hosted на VPS)
- Причина: есть VPS, нужен full-stack (события + сессии + фичефлаги)
- Установка: Docker на отдельном VPS

### Дашборд
[структура из шаблона, адаптированная под проект]

### Алерты
[из таблицы, адаптированные]

### Задачи на интеграцию (→ tasks.md)
- [ ] Установить PostHog на VPS (Docker)
- [ ] Интегрировать SDK в фронтенд
- [ ] Настроить события: регистрация, onboarding, покупка
- [ ] Создать дашборд с ключевыми метриками

```

## Приоритет при конфликтах

1. Приватность данных > полнота аналитики (GDPR/152-ФЗ)
2. Self-hosted предпочтительнее при наличии VPS (контроль данных)
3. Лучше меньше метрик, но отслеживаемых → чем 50 метрик, на которые не смотришь
4. Алерты должны быть actionable — не спам, а сигнал к действию
5. North Star Metric — одна. Не три, не пять. Одна.
