# Завершение фичи — /done

Выполнить когда ФИЧА ПОЛНОСТЬЮ ГОТОВА (все задачи выполнены, всё задеплоено и проверено).
Отличается от /end (завершение сессии) — /done закрывает фичу целиком.

---

## Шаги:

### 1. Финальный DoD (Definition of Done)

Проверить перед закрытием:
- [ ] Все задачи в docs/tasks.md отмечены [x]
- [ ] Все тесты зелёные (pytest / jest / etc)
- [ ] Деплой выполнен и проверен (health check, smoke test)
- [ ] Anti-mirage check пройден на финальном коде
- [ ] Нет console.log / print() для отладки в коде

Если что-то не готово — НЕ закрывать, сообщить пользователю.

### 2. Обновить Project Knowledge

Главная ценность /done — актуализация "живых" документов проекта.

**docs/project-knowledge/architecture.md:**
- Обновить структуру проекта (новые модули, компоненты)
- Обновить модель данных (новые сущности, поля)
- Обновить стек (новые зависимости)
- Обновить схему компонентов

**docs/project-knowledge/patterns.md:**
- Добавить новые конвенции если сложились
- Обновить паттерны обработки ошибок если изменились
- Обновить тестовые практики

**docs/project-knowledge/deployment.md:**
- Обновить env vars (новые переменные)
- Обновить Docker/Nginx конфиг если менялся
- Обновить порты/домены

**docs/project-knowledge/project.md:**
- Обновить scope (что теперь умеет продукт)
- Обновить статус фич (что готово)

### 3. Обновить DECISIONS.md

Добавить стратегические решения принятые во время фичи (если ещё не добавлены через /end).

### 4. Обновить AI_NOTES последнего блока

Финальные заметки: итог фичи, что работает, известные ограничения.

### 5. Записать session log

Дополнить docs/sessions/YYYY-MM-DD.md финальной записью о завершении фичи.

### 6. Архивировать артефакты (для L размера)

Для больших фич — переместить спеки в архив:
```bash
mkdir -p docs/archive/[feature-name]/
cp docs/user-spec.md docs/archive/[feature-name]/
cp docs/tech-spec.md docs/archive/[feature-name]/
cp docs/tasks.md docs/archive/[feature-name]/
cp docs/brainstorm.md docs/archive/[feature-name]/
```

Для M размера — пропустить, файлы перезапишутся следующей фичей.

### 7. Обновить README.md

Если фича добавляет пользовательский функционал:
- Обновить описание проекта
- Обновить список фич
- Обновить инструкции по запуску если изменились

### 8. Git commit + push

```bash
git add -A
git commit -m "feat: [название фичи] — complete"
git push
```

### 9. Показать итог

```
━━━ 🏁 ФИЧА ЗАВЕРШЕНА ━━━
Фича: [название]
Задач: [X выполнено из X]
Коммитов: [N]
Обновлено: architecture.md, patterns.md, deployment.md
Время: ~[часы]

Что дальше: [следующая фича / бэклог]
```
