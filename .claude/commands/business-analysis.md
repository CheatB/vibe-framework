
# /business-analysis — Бизнес-анализ продукта

Standalone-команда для запуска бизнес-анализа вне `/new-project` pipeline.
Вызывает те же 5 блоков из Фазы 1.8.

---

## Когда использовать

- Продукт уже существует, но бизнес-составляющая не проработана
- Нужен отдельный анализ одного или нескольких блоков
- Пользователь говорит "проанализируй бизнес" / "нужна монетизация" / "как выводить на рынок"

## Workflow

```
1. Спросить пользователя: какие блоки нужны?
   (или запустить все 5 по умолчанию)

2. Для каждого выбранного блока:
   a. Загрузить скилл
   b. Задать уточняющие вопросы (батч 3-5)
   c. Сформировать результат

3. Записать в docs/business-analysis.md
4. Показать итог
```

## Блоки

| # | Блок | Скилл | Триггеры |
|---|------|-------|----------|
| 1 | Legal Compliance | `legal-compliance` | ПД, платежи, оферта, закон, юр |
| 2 | Monetization | `monetization` | монетизация, деньги, тарифы, оплата |
| 3 | GTM | `gtm` | рынок, конкуренты, продвижение, лендинг |
| 4 | Analytics | `analytics-setup` | метрики, аналитика, дашборд |
| 5 | Accessibility | `accessibility` | доступность, a11y, контраст |

## Формат вывода

```markdown
# Бизнес-анализ: [Название проекта]

## 1. Legal Compliance
[результат скилла]

## 2. Monetization
[результат скилла]

## 3. GTM
[результат скилла]

## 4. Analytics
[результат скилла]

## 5. Accessibility
[результат скилла]

## Итого: задачи на реализацию
- [ ] [задача из legal]
- [ ] [задача из monetization]
- [ ] [задача из gtm]
- [ ] [задача из analytics]
- [ ] [задача из accessibility]
```

## Приоритет блоков

Если пользователь не указал конкретный блок — определить автоматически:

1. **Есть платежи/регистрация** → Legal + Monetization обязательны
2. **Новый продукт** → GTM обязателен
3. **Есть UI** → Accessibility обязателен
4. **Любой продукт** → Analytics рекомендован
