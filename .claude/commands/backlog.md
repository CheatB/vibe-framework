# /backlog — Управление бэклогом проекта

Команда для работы с бэклогом: просмотр, приоритизация, добавление и выбор задач.

## Использование

```
/backlog              — показать текущий бэклог
/backlog add          — добавить задачу в бэклог
/backlog next         — выбрать следующую задачу для работы
/backlog prioritize   — пересортировать бэклог по приоритетам
```

## Формат бэклога (backlog.md)

```markdown
# Бэклог проекта

## 🔥 Критичные (блокеры)
- [ ] [#ID] Описание задачи — оценка: S/M/L

## 📋 Высокий приоритет
- [ ] [#ID] Описание задачи — оценка: S/M/L

## 📝 Средний приоритет
- [ ] [#ID] Описание задачи — оценка: S/M/L

## 💡 Низкий приоритет (nice to have)
- [ ] [#ID] Описание задачи — оценка: S/M/L

## ✅ Выполнено
- [x] [#ID] Описание — дата завершения
```

## /backlog (без аргументов)

1. Найди `backlog.md` в корне проекта (или путь из CLAUDE.md)
2. Покажи текущее состояние:
   - Сколько задач всего / по приоритетам
   - Что в работе сейчас
   - Топ-3 следующих задач
3. Если бэклога нет — предложи создать

## /backlog add

1. Спроси у пользователя:
   - Что за задача? (описание)
   - Приоритет? (критичный / высокий / средний / низкий)
   - Оценка размера? (S / M / L)
2. Присвой следующий свободный #ID
3. Добавь в соответствующую секцию бэклога
4. Покажи результат

Если пользователь описал задачу в свободной форме — определи приоритет и размер сам, покажи для подтверждения.

## /backlog next

1. Прочитай бэклог
2. Выбери задачу с наивысшим приоритетом, которая:
   - Не заблокирована другими задачами
   - Не в работе
   - Подходит по размеру (начни с S/M если есть)
3. Покажи пользователю:
   ```
   Следующая задача: [#ID] Описание
   Приоритет: высокий | Размер: M
   Начинаем?
   ```
4. При подтверждении — отметь задачу как "в работе" и начни выполнение

## /backlog prioritize

1. Прочитай весь бэклог
2. Предложи пересортировку по критериям:
   - Бизнес-ценность (что даст пользователям)
   - Зависимости (что разблокирует другие задачи)
   - Усилие (quick wins сначала)
3. Покажи предложенный порядок
4. Примени после одобрения пользователя

## Связь с Deep Interview

Если при /new-project пользователь не ответил на все вопросы Deep Interview:
- Неотвеченные вопросы автоматически попадают в бэклог как задачи с тегом `[вопрос]`
- При /backlog next — вопросы показываются в контексте связанных задач

## Интеграция с /done

При выполнении /done (завершение фичи):
1. Отметить задачу как выполненную в бэклоге
2. Перенести в секцию "✅ Выполнено" с датой
3. Если задача породила новые задачи — добавить их в бэклог
