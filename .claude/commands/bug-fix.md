# /bug-fix — Анализ и автоисправление ошибок

Проанализируй ошибку и исправь её.

## Вход
Пользователь вставляет:
- Stack trace / traceback
- Сообщение об ошибке
- Логи
- Или описание бага

## Алгоритм

### 1. Анализ ошибки
```
━━━ 🐛 BUG-FIX: Анализ ━━━
Тип: [TypeError / ImportError / RuntimeError / ...]
Файл: [путь]
Строка: [номер]
Причина: [краткое описание]
```

### 2. Найди корень проблемы
- Прочитай файл с ошибкой
- Найди контекст: вызывающий код, зависимости
- Если ошибка в зависимости — проверь версии в package.json / requirements.txt
- Проверь git log — когда сломалось (`git log --oneline -20`)

### 3. Исправь
- Внеси минимальное исправление (не рефактори весь файл!)
- Если нужно — добавь тест который воспроизводит баг
- Запусти тесты чтобы убедиться что:
  a. Баг исправлен
  b. Ничего другого не сломалось

### 4. Отчёт
```
━━━ 🐛 BUG-FIX: Готово ━━━
Причина: [что было не так]
Исправление: [что сделал]
Файлы: [список изменённых файлов]
Тесты: [passed/failed]
```

## Правила
- МИНИМАЛЬНОЕ исправление — не трогай то что работает
- Если не уверен в причине — спроси пользователя
- Если баг в сторонней библиотеке — предложи workaround
- Добавь комментарий в коде: `# FIX: описание бага и исправления`
