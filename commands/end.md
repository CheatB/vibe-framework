# Завершение рабочей сессии

Выполнить ПРИ КАЖДОМ завершении работы над проектом (перед Ctrl+D).

## Шаги:

### 1. Обновить AI_NOTES текущего блока

Если работали над блоком задач — обновить или создать `docs/ai-notes/block-N.md`:
- Что сделано
- Решения и ПОЧЕМУ
- Контракты между модулями
- Риски и техдолг
- Как проверить
- Контекст для следующего блока

### 2. Обновить DECISIONS.md

Если были приняты стратегические решения (выбор технологии, архитектурный подход, модель данных) — добавить ADR в `docs/DECISIONS.md`:

```markdown
## ADR-NNN: [Название]
**Дата:** YYYY-MM-DD
**Статус:** принято
**Контекст:** [Проблема]
**Решение:** [Что выбрали]
**Альтернативы:**
- [A] — почему нет
**Последствия:** [Что это значит]
```

Если стратегических решений не было — пропустить.

### 3. Записать лог сессии

Создать или дополнить `docs/sessions/YYYY-MM-DD.md`:

```markdown
# Сессия YYYY-MM-DD

## Что сделано
- [конкретно что реализовано / исправлено / настроено]

## Решения
- [ссылки на новые ADR, если были]

## Следующие шаги
- [что делать в следующей сессии]
- [какой блок / задача следующий]

## Git
- Коммиты: [хеши]
```

Если за один день несколько сессий — дополнять файл, не перезаписывать.

### 4. Обновить tasks.md

Отметить выполненные задачи галочками [x]. Обновить DoD блока если завершён.

### 5. Git commit

```bash
git add -A
git commit -m "session: [краткое описание что сделано]"
git push
```

### 6. Показать итог пользователю

Кратко:
- Что сделано за сессию
- Текущий прогресс (X из Y задач)
- Что делать дальше
