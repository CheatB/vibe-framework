# Команда: Code Review

## Что делает
Проводит ревью кода на качество, безопасность, производительность

## Как использовать
Пользователь пишет: `/review` или `/code-review`

## Что проверять

### 1. Безопасность
- ❌ Захардкоженные секреты (API ключи, пароли)
- ❌ SQL инъекции
- ❌ XSS уязвимости
- ❌ Небезопасные зависимости

### 2. Качество кода
- ❌ Дублирование кода (DRY principle)
- ❌ Слишком длинные функции (>50 строк)
- ❌ Слишком сложные условия (циклометрическая сложность)
- ❌ Неиспользуемый код
- ❌ Плохие названия переменных (a, b, x, temp)

### 3. Производительность
- ❌ Лишние циклы
- ❌ Неоптимальные запросы к БД
- ❌ Memory leaks

### 4. Тесты
- ❌ Отсутствие тестов
- ❌ Покрытие < 80%
- ❌ Тесты не проходят

### 5. TypeScript (если применимо)
- ❌ Использование `any`
- ❌ Отсутствие типов
- ❌ Ошибки типов

## Формат вывода
```
📋 Code Review Results:

🔴 CRITICAL (должно быть исправлено):
1. src/api.ts:15 - Захардкожен API ключ
2. src/auth.ts:42 - SQL инъекция

🟡 WARNING (рекомендуется исправить):
1. src/utils.ts:88 - Функция слишком длинная (120 строк)
2. src/components/Form.tsx - Отсутствуют тесты

🟢 GOOD:
- Хорошее покрытие тестами (85%)
- Код хорошо типизирован
- Понятные названия переменных

Общая оценка: 6/10
```
