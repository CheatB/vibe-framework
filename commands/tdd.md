# Команда: TDD (Test-Driven Development)

## Что делает
Помогает писать код через тесты (сначала тест, потом код)

## Как использовать
Пользователь пишет: `/tdd` или `/test-first`

## Workflow

### 1. Понять задачу
- Спроси: "Что должен делать код?"
- Уточни входные данные и ожидаемый результат

### 2. Написать тест ПЕРВЫМ
```typescript
// Сначала пиши тест!
describe('calculateDiscount', () => {
  it('should apply 10% discount', () => {
    expect(calculateDiscount(100, 10)).toBe(90);
  });
});
```

### 3. Запустить тест (он должен упасть)
- Убедись что тест падает (RED)
- Это доказывает, что тест рабочий

### 4. Написать минимальный код
```typescript
function calculateDiscount(price: number, percent: number) {
  return price - (price * percent / 100);
}
```

### 5. Запустить тест (он должен пройти)
- Тест зелёный (GREEN)

### 6. Рефакторинг
- Улучши код, не ломая тест
- Проверь что тесты всё ещё проходят

## Цикл: RED → GREEN → REFACTOR

## Пример
```
Пользователь: /tdd Нужна функция для валидации email

Ты:
📝 TDD режим активирован

Шаг 1: Пишу тест
✅ Создан test/validators.test.ts

Шаг 2: Запускаю тест
❌ FAILED (как и ожидалось)

Шаг 3: Пишу код
✅ Создан src/validators.ts

Шаг 4: Запускаю тест
✅ PASSED

Готово! Хочешь добавить edge cases?
```
