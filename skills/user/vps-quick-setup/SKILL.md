# VPS Quick Setup Skill

**Быстрая настройка нового VPS на Beget с автоматическим добавлением в Claude.**

## 🎯 Когда использовать

**Триггеры:**
- "настрой новый vps"
- "добавь vps"
- "/setup-vps"
- "подключи vps"

**Используй когда:**
- Создал новый VPS на Beget
- Нужно быстро добавить его в Claude
- Хочешь избавиться от проблем с паролями

### ШАГ 2: Создание init скрипта

```

✓ Создан: /tmp/beget-vps-init.sh
✓ Скрипт настроит:
  - Пользователя claude
  - SSH доступ по ключу
  - Sudo без пароля
  - Отключение доступа по паролю
  - Базовые пакеты (git, python3, nodejs)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🎯 ИНСТРУКЦИЯ ДЛЯ ТЕБЯ:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1️⃣ Создай VPS на Beget (если ещё не создал)
   → Запиши IP адрес

2️⃣ Открой веб-консоль в панели Beget:
   Панель → Твой VPS → Кнопка "Консоль"

3️⃣ В веб-консоли залогинься:
   Login: root
   Password: [пароль из email от Beget]

4️⃣ Скопируй и запусти ЭТУ КОМАНДУ:

bash <(curl -s https://gist.githubusercontent.com/.../init.sh)

   ИЛИ если нет curl:

bash <(wget -qO- https://gist.github.com/.../init.sh)

   ИЛИ вручную (вариант 100% работающий):

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[Показываю полный текст скрипта для копипасты]

```bash
#!/bin/bash
set -e

echo "🚀 Beget VPS Quick Setup"
echo "━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━"

# 1. Обновление системы
echo "📦 Обновление системы..."
apt-get update -qq
apt-get upgrade -y -qq

# 2. Установка базовых пакетов
echo "📦 Установка пакетов..."
apt-get install -y -qq curl git python3 python3-pip nodejs npm

# 3. Создание пользователя claude
echo "👤 Создание пользователя claude..."
if id "claude" &>/dev/null; then
    echo "  ✓ Пользователь claude уже существует"
else
    useradd -m -s /bin/bash claude
    echo "  ✓ Пользователь claude создан"
fi

# 4. Настройка SSH
echo "🔐 Настройка SSH доступа..."
mkdir -p /home/claude/.ssh
chmod 700 /home/claude/.ssh

# ВАЖНО: Замени на свой публичный ключ!
cat >> /home/claude/.ssh/authorized_keys << 'EOF'
[ТВОЙ_ПУБЛИЧНЫЙ_КЛЮЧ_СЮДА]
EOF

chmod 600 /home/claude/.ssh/authorized_keys
chown -R claude:claude /home/claude/.ssh

# 5. Sudo без пароля
echo "🔓 Настройка sudo..."
echo "claude ALL=(ALL) NOPASSWD:ALL" > /etc/sudoers.d/claude
chmod 440 /etc/sudoers.d/claude

# 6. Отключение SSH по паролю (безопасность)
echo "🔒 Отключение доступа по паролю..."
sed -i 's/#PasswordAuthentication yes/PasswordAuthentication no/' /etc/ssh/sshd_config
sed -i 's/PasswordAuthentication yes/PasswordAuthentication no/' /etc/ssh/sshd_config
systemctl restart sshd

# 7. Получение IP
MY_IP=$(curl -s ifconfig.me 2>/dev/null || curl -s icanhazip.com 2>/dev/null || echo "UNKNOWN")

echo ""
echo "━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━"
echo "✅ VPS настроен успешно!"
echo "━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━"
echo ""
echo "📋 IP адрес этого VPS: $MY_IP"
echo ""
echo "🧪 Тестирование подключения:"
echo "   ssh claude@$MY_IP"
echo ""
echo "📝 Скопируй этот IP и вернись в Claude Desktop!"
echo ""
```

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

5️⃣ После выполнения скрипта:
   → Скопируй IP адрес который показал скрипт
   → Вернись сюда и напиши: VPS готов, IP: [твой IP]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 🔧 ВАЖНЫЕ МЕХАНИКИ

### Idempotent скрипт

Скрипт можно запускать **много раз** — он не сломает VPS:

```bash
# Первый запуск
bash init.sh  # Всё настроит

# Второй запуск (случайно)
bash init.sh  # Проверит что уже настроено, обновит если надо

# Третий запуск
bash init.sh  # Снова безопасно
```

---

### Автоматическая замена публичного ключа

При генерации скрипта Claude **автоматически вставляет** твой публичный ключ:

```bash
# НЕ нужно вручную редактировать!
# Claude уже вставил твой ключ в скрипт
cat >> /home/claude/.ssh/authorized_keys << 'EOF'
ssh-ed25519 AAAAC3NzaC1lZDI1... [твой реальный ключ]
EOF
```

---

### Безопасность

**Что делает скрипт для безопасности:**

1. ✅ Отключает доступ по паролю (только SSH ключи)
2. ✅ Создаёт отдельного пользователя (не root)
3. ✅ Sudo без пароля только для claude
4. ✅ Правильные права на .ssh директорию (700)
5. ✅ Правильные права на authorized_keys (600)

## 📋 TROUBLESHOOTING

### "Permission denied" при SSH

**Проблема:** После скрипта не могу зайти по SSH

**Решение:**
```bash
# 1. Проверь что ключ добавлен в SSH agent
ssh-add ~/.ssh/beget_vps_key

# 2. Проверь подключение с явным указанием ключа
ssh -i ~/.ssh/beget_vps_key claude@194.113.209.218

# 3. Если не работает — используй веб-консоль
# и проверь:
cat /home/claude/.ssh/authorized_keys
# Твой ключ там должен быть
```

---

### Скрипт не выполнился до конца

**Проблема:** Скрипт прервался посередине

**Решение:**
```bash
# Просто запусти ещё раз!
bash init.sh

# Скрипт idempotent — не сломает ничего
```

---

### Забыл добавить публичный ключ в скрипт

**Проблема:** Запустил скрипт, но забыл вставить свой ключ

**Решение:**
```bash
# Через веб-консоль добавь ключ вручную:
mkdir -p /home/claude/.ssh
echo "ssh-ed25519 AAAAC3..." >> /home/claude/.ssh/authorized_keys
chmod 600 /home/claude/.ssh/authorized_keys
chown -R claude:claude /home/claude/.ssh
```

### Один ключ для всех VPS

**Рекомендую:**
```
Создай один ключ: ~/.ssh/beget_vps_key
Используй его для ВСЕХ VPS на Beget
Не нужно создавать ключ для каждого VPS
```

**Преимущества:**
- Проще управлять
- Один ключ в SSH agent
- Легче делать бэкап

---

### Backup SSH ключа

**КРИТИЧНО: Сделай бэкап приватного ключа!**

```bash
# Скопируй приватный ключ в безопасное место
cp ~/.ssh/beget_vps_key ~/Dropbox/Backups/
# или
cp ~/.ssh/beget_vps_key /Volumes/USB-Drive/

# Если потеряешь — не сможешь зайти на VPS!
```

## 🔧 ДОПОЛНИТЕЛЬНЫЕ СЦЕНАРИИ

### Сценарий 1: Проблемы с root паролем

```
Ты: /setup-vps
[Claude создаёт скрипт]

Ты: Пароль root не работает

Claude:
😕 Понятно, такое часто бывает на Beget.

🎯 У тебя есть 3 варианта:

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
ВАРИАНТ A: Веб-консоль (VNC) ⭐ РЕКОМЕНДУЮ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. В панели Beget найди свой VPS
2. Найди кнопку "Консоль" или "VNC" или "Терминал"
3. Откроется консоль в браузере
4. Там попробуй залогиниться:
   login: root
   password: [пароль из email]
5. Если и там не работает → ВАРИАНТ B

💡 Веб-консоль обычно лучше обрабатывает копипасту!

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
ВАРИАНТ B: Поддержка Beget
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Напиши в поддержку Beget:

"Здравствуйте! Не могу зайти по SSH на VPS [твой IP].
Пробовал пароль из email — не подходит.
Можете сбросить пароль или помочь разобраться?"

Обычно отвечают быстро.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
ВАРИАНТ C: Добавление ключа через другой VPS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Если у тебя есть другой рабочий VPS на том же аккаунте:

1. Зайди на другой VPS
2. Оттуда зайди на новый VPS (если это возможно)
3. Добавь SSH ключ

Но это сложно, лучше ВАРИАНТ A или B.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Какой вариант попробуешь?
```

## 💡 BEST PRACTICES

### Один универсальный ключ для всех VPS

**Преимущества:**
```
✅ Проще управлять
✅ Один ключ в SSH agent
✅ Легче делать бэкап
✅ Один конфиг для всех VPS
✅ Проще добавлять новые VPS
```

**Как использовать:**
```
1. Создал универсальный ключ один раз
2. Использую его для ВСЕХ новых VPS
3. Мигрировал старые VPS на него
4. Удалил старые ключи (через месяц)
```

---

### Бэкап SSH ключей

**КРИТИЧНО: Сделай бэкап приватного ключа!**

```bash
# Создай папку для бэкапов
mkdir -p ~/Backups/ssh-keys

# Скопируй приватный ключ
cp ~/.ssh/beget_vps_key ~/Backups/ssh-keys/
cp ~/.ssh/beget_vps_key.pub ~/Backups/ssh-keys/

# Сделай архив с датой
tar -czf ~/Backups/ssh-keys-$(date +%Y%m%d).tar.gz ~/Backups/ssh-keys/

# Загрузи в облако (Dropbox/Google Drive/etc)
```

**Если потеряешь ключ:**
- ❌ Не сможешь зайти на VPS
- ❌ Придётся пересоздавать доступ через веб-консоль
- ❌ Потеряешь время

---

### Проверка ключей

**Периодически проверяй:**

```bash
# Какие ключи на VPS
ssh claude@IP 'cat ~/.ssh/authorized_keys'

# Тестируй подключение
ssh -i ~/.ssh/beget_vps_key claude@IP 'echo OK'

# Проверяй права
ssh claude@IP 'ls -la ~/.ssh/'
```
